# Этап 1. Архитектура проекта GetBox

## Общая концепция

GetBox — это распределенная система управления ячейками постаматов, построенная на базе ESP32 и MQTT.

Понятие  компонента **Box (GetBox)** является исключительно **логической сущностью на уровне программного обеспечения** и не представляет собой отдельное физическое устройство.

Один Box объединяет несколько ESP32, которые работают совместно и подключаются к единому MQTT-брокеру через Wi-Fi.

---

## Основные архитектурные принципы

- Все ESP32 подключаются к существующей Wi-Fi сети через роутер.
- Все ESP32 подключаются к одному MQTT-брокеру (Mosquitto).
- MQTT выступает только транспортным механизмом доставки сообщений.
- FastAPI содержит всю бизнес-логику маршрутизации команд.
- FastAPI хранит информацию о том:
  - какой Box существует;
  - какие ESP32 входят в состав Box;
  - какие ячейки обслуживает каждая ESP32;
  - какие GPIO соответствуют каждой ячейке.
- ESP32 знают только о своих локальных ресурсах:
  - собственном Controller ID;
  - своих GPIO;
  - закрепленных за ними ячейках.
- ESP32 не взаимодействуют друг с другом напрямую.
- Box используется исключительно как программный идентификатор группы устройств.

# Компоненты системы

| Компонент | Роль в системе | Вход | Выход | Протокол | Ответственность |
|---|---|---|---|---|---|
| Клиентская система заказчика | Внешняя система, инициирующая работу постамата | Действия пользователя, заказ, подтверждение оплаты | HTTP-запросы в API | HTTPS / REST / JSON | Не взаимодействует напрямую с ESP32 и MQTT |
| FastAPI Server | Центральная бизнес-логика системы | HTTP-запросы клиентов, MQTT-события | MQTT-команды, HTTP-ответы | HTTP + MQTT | Проверяет права, определяет Box и нужную ESP32, маршрутизирует команды |
| Mosquitto MQTT Broker | Центр обмена сообщениями | MQTT-сообщения от FastAPI и ESP32 | MQTT-сообщения подписчикам | MQTT | Передает сообщения между сервером и устройствами |
| Wi-Fi Router | Сетевая инфраструктура | Подключения ESP32 | Доступ к MQTT-серверу | Wi-Fi / TCP/IP | Обеспечивает связь ESP32 с сервером |
| Box / GetBox | Логическая сущность системы | Команды FastAPI | События и статусы устройств | MQTT over Wi-Fi | Объединяет несколько ESP32 в один программный объект |
| ESP32 Sensors Controller | Контроллер датчиков внутри Box | 3 DHT11 | Данные мониторинга | GPIO + MQTT | Считывает температуру и влажность и передает данные на сервер |
| DHT11 | Датчик температуры и влажности | Температура и влажность окружающей среды | Цифровой сигнал | Data GPIO | Используется только для мониторинга состояния внутри Box |
| ESP32 Cell Controller | Контроллер группы ячеек | MQTT-команды, сигналы герконов | Управление реле, статусы | MQTT + GPIO | Управляет 4–5 ячейками |
| Реле-модуль | Силовой исполнительный модуль | GPIO-сигнал ESP32 | Подача питания 12V | GPIO / DC | Разделяет слаботочную и силовую части |
| Замок ячейки | Исполнительный механизм | Питание 12V | Физическое открытие двери | DC 12V | Открывает ячейку |
| Геркон | Контроль состояния двери | Положение двери | HIGH / LOW сигнал | GPIO IN | Определяет открытие, закрытие и возможное вскрытие |
| Блок питания 12V | Питание замков | 220V или внешний источник | 12V | DC | Питает электромеханические замки |
| Блок питания ESP32 | Питание контроллеров | USB / 5V | 5V / 3.3V | DC | Питает ESP32 и периферию |

# Структура одного Box

```text
Box_003
│
├── ESP32 Sensors Controller
│   ├── DHT11 #1
│   ├── DHT11 #2
│   └── DHT11 #3
│
├── ESP32 Cell Controller #1
│   ├── Cell 1
│   ├── Cell 2
│   ├── Cell 3
│   ├── Cell 4
│   └── Cell 5
│
├── ESP32 Cell Controller #2
│   ├── Cell 6
│   ├── Cell 7
│   ├── Cell 8
│   ├── Cell 9
│   └── Cell 10
│
└── ESP32 Cell Controller #N
    └── Дополнительные ячейки
```

Количество Cell Controller внутри Box не ограничено и определяется количеством ячеек конкретного постамата.

# Аппаратная схема одной ячейки

```text
ESP32 GPIO OUT
        │
        ▼
      Реле
        │
        ▼
    Питание 12V
        │
        ▼
      Замок


Геркон
   │
   ▼
ESP32 GPIO IN
```

# Формула подключения GPIO

Каждая ячейка использует два GPIO.

```text
1 ячейка
=
1 GPIO OUT (реле)
+
1 GPIO IN (геркон)
```

Примеры:

| Количество ячеек | Необходимые GPIO |
|---|---:|
| 1 | 2 |
| 4 | 8 |
| 5 | 10 |

Практическое ограничение:

```text
1 ESP32 Cell Controller
≈ 4–5 ячеек
```

# Принцип маршрутизации команд

Пример команды:

```text
Открыть ячейку №6 в Box_003
```

FastAPI выполняет следующие действия:

```text
1. Получает запрос клиента.

2. Определяет:
   Box = Box_003

3. Находит:
   Cell 6 → Controller ctrl_02

4. Публикует MQTT-команду
   для ctrl_02.

5. ESP32 ctrl_02 активирует
   соответствующее реле.

6. Геркон подтверждает
   изменение состояния двери.

7. ESP32 отправляет статус обратно.
```
